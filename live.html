ChatGPT said: Yes — this is a known issue that happens with Songkick widgets/embeds when the user clicks an event and Songkick receives an HTTP request without a proper referrer or with certain tracking parameters stripped. Many browsers, ad-blockers, and privacy settings now block referrers on iframe → top-level navigations, which causes Songkick to think the request is suspicious and return 403 Forbidden. Refreshing fixes it because the user is now already on songkick.com directly, so the referrer is no longer blocked. Try adding: sandbox="allow-scripts allow-same-origin allow-top-navigation-by-user-activation" Full example: This tells the browser: “Yes, the user is allowed to click a link and navigate out of the iframe.” 3. Make sure the embed URL is HTTPS Songkick blocks mixed-content or downgraded requests. If your site is HTTPS but your embed uses HTTP, it can cause 403 on click. 4. Wrap the widget link in a redirect If the 403 continues due to strict browser privacy settings, you can create a workaround such as: Your link → your site → Songkick.
